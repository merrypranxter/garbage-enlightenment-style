# APP INTEGRATION NOTES
This file explains how Garbage Enlightenment can be implemented or exposed inside apps,
style engines, shader tools, prompt builders, or generative systems.
The goal is not only to describe the style.
The goal is to make it operable.
## CORE APP MODEL
This style works well as a modular system with linked but separable subsystems.
Recommended major modules:
- body glyph module
- Sweet Corruption module
- mock esoterica module
- creature module
- abstract hallucination field module
- motion / metamorphosis module
- anti-drift module
## SUGGESTED CONTROL KNOBS
### 1. Cute ↔ Gross
Controls how strongly sweet motifs are contaminated by body weirdness.
Range examples:
- mostly cute
- cute with slight contamination
- equal mix
- gross-funny
- harsh freak mode
### 2. Sacred ↔ Parody
Controls how serious or ridiculous the symbolic/esoteric charge feels.

Range examples:
- almost sincere
- fake revelation
- parody occult
- stupid sacred nonsense
### 3. Figurative ↔ Abstract
Controls how image space distributes between:
- readable creatures/faces/glyphs
- contour bands / plasma / abstract signal soup
### 4. Stable ↔ Mutating
Controls how much identity instability is visible in the output.
### 5. Pathetic ↔ Ominous
Controls emotional posture of creatures and icons.
### 6. Cheap ↔ Refined
Should never go fully refined.
This knob should be constrained.
Even the “refined” end should keep some anti-polish energy.

## LSD-LOGIC EMPHASIS NOTE
A user-facing “LSD emphasis” control should be described carefully.
It should not promise:
- literal trip realism
- accurate psychedelic phenomenology
- medical or documentary fidelity
It should instead mean something like:
- higher symbolic instability
- more morph pressure
- stronger figure-pattern leakage
- faster meaning reassignment
- more over-significant emergence logic
In other words:
**cartoonized LSD-hallucination structure**, not literal simulation.
## SAFETY / TONE IMPLEMENTATION NOTE

The style benefits from weirdness, body symbolism, and mild horror, but its default engine
should preserve:
- play
- color
- symbolic humor
- non-realistic treatment
- anti-grim flexibility
That keeps it from collapsing into generic gore or oppressive darkness.
## SYSTEM PRIORITIES
If building style logic into an app, prioritize these in order:
1. body glyph recognition / generation
2. mutational continuity
3. figure-pattern permeability
4. tonal friction
5. anti-polish line logic
6. mock-esoteric contamination
7. Sweet Corruption balance
8. palette specificity
## FINAL RULE
Do not build the app around:
“make psychedelic art.”
Build it around:
**make symbolic cartoon mutation systems with body glyphs, Sweet Corruption, fake revelation,
and cheap hallucination fields**
