# REPO MAP
This document explains the structure of the Garbage Enlightenment repo and what each file is
for.
## Core identity files
- `README.md`
- top-level explanation of the repo, what the style is, what it is for, and how to use it
- `STYLE_OVERVIEW.md`
- plain-language definition of the style and its major identity traits
- `STYLE_PILLARS.md`
- the core laws of the style; the main structural anchors that must remain intact

- `STYLE_BOUNDARIES.md`
- what belongs inside the style and what falls outside it
- `ANTI_DRIFT.md`
- common failure modes and how to correct them
## Tone and emotional files
- `EMOTIONAL_TONE.md`
- the style’s emotional spectrum, tonal friction, and mood logic
- `SWEET_CORRUPTION.md`
- the cute ↔ gross axis and how sweetness and contamination coexist
## Symbol and content files
- `BODY_GLYPHS.md`
- body parts used as symbols, emblems, stickers, portals, and mutation sites
- `MOCK_ESOTERICA.md`
- parody-occult, pseudo-sacred, conspiratorial, and fake-revelation imagery
- `CREATURE_ARCHETYPES.md`
- recurring creature types and how beings should behave inside the style
- `FACE_AND_HEAD_MUTATIONS.md`
- how faces and heads mutate, split, recur, and hold symbolic instability
- `MOTIF_LIBRARY.md`
- recurring motifs and symbol families used throughout the style
## Formal system files
- `COMPOSITION_GRAMMAR.md`
- rules for staging, layout, density, symbolic emphasis, and figure-pattern balance
- `LINEWORK_AND_SIMPLICITY.md`
- rules for outlines, flatness, graphic clarity, and anti-polish drawing logic
- `PALETTE_SYSTEM.md`
- native palette behavior, color family roles, and palette drift prevention
## Motion and abstraction files

- `MOTION_AND_LOOP_BEHAVIOR.md`
- rules for animation, morphing, symbolic substitution, and loop cadence
- `ABSTRACT_HALLUCINATION_FIELDS.md`
- contour-band fields, thermal melts, plasma, signal soup, and non-figurative subsystem logic
- `GLSL_SHADER_TRANSLATIONS.md`
- procedural/shader translation of the style into field logic, emergence behavior, and system
parameters
## Prompt utility files
- `PROMPT_TEMPLATES.md`
- full prompt formulas for stills, creatures, icons, animations, and shader prompts
- `PROMPT_SNIPPETS.md`
- modular style chunks for quick prompt building
- `IMAGE_GEN_RECIPES.md`
- practical still-image prompting recipes
- `ANIMATION_RECIPES.md`
- practical animation / loop prompting recipes
## Machine-readable file
- `style_spec.json`
- structured summary of the style for app integration, tooling, or future machine-readable logic
## Suggested workflow
### If you are new to the repo
Start with:
1. `README.md`
2. `STYLE_OVERVIEW.md`
3. `STYLE_PILLARS.md`
4. `STYLE_BOUNDARIES.md`
5. `ANTI_DRIFT.md`
### If you are prompt-building
Use:
1. `PROMPT_TEMPLATES.md`
2. `PROMPT_SNIPPETS.md`

3. `IMAGE_GEN_RECIPES.md`
4. `ANIMATION_RECIPES.md`
### If you are designing systems / apps / shaders
Use:

2. `GLSL_SHADER_TRANSLATIONS.md`

4. `MOTION_AND_LOOP_BEHAVIOR.md`
### If you are refining imagery
Use:
1. `BODY_GLYPHS.md`
