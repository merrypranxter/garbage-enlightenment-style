# GENERATIVE SYSTEM NOTES

This document describes how Garbage Enlightenment behaves as a generative system rather
than just a static style reference.

This is important because the style is not just a look.
It is a behavior set.

## Core principle

Garbage Enlightenment should be treated as a **mutation grammar**.

It is generated most successfully when the system understands:
- symbol families
- body-part roles
- contamination logic
- motion logic
- tonal mixtures
- figure-pattern permeability
- anti-drift constraints

Not just:
- color palette
- outline style
- “trippy” vibes

## 2. SWEET CORRUPTION ENGINE

This engine mixes sweetness with body weirdness or symbolic instability.

### Input families
- stars
- hearts
- rainbows
- dolphins
- unicorn-ish forms
- clouds
- bubbles

- toy motifs

### Corruption methods
- eye insertion
- mouth insertion
- asymmetry
- mutation
- contour bleed
- occult contamination
- emotional wrongness
- creature fusion

### Output effect
Prevents the image from becoming too grim or too pure.

---

## 3. MOCK ESOTERICA ENGINE

This engine handles:
- eyes in pyramids
- portals
- fake sigils
- aura rings

- symbolic staging
- revelation graphics

### Core rule
The symbol must retain some charge while also being:
- ridiculous
- bodily
- synthetic
- low-fi
- fake-serious

### Output danger
If not constrained, this engine drifts into:
- solemn occult branding
- over-clean icon systems
- mystical luxury design

So body contamination and anti-cool bias are crucial.

---

## 4. ABSTRACT HALLUCINATION FIELD ENGINE

This engine builds:

- contour bands
- thermal maps
- plasma blobs
- signal soup
- false topology
- emergence substrate

### Purpose
Not just “background.”
This engine should feel like:
- an almost-body
- a pre-symbol
- a potential creature zone
- a fake cosmic weather system

### Important behavior
It should maintain emergence pressure.
It should feel like a face or glyph could appear at any moment.

---

## 5. METAMORPHIC TRANSITION ENGINE

This is the motion grammar engine.

### Responsibilities
- contour inheritance
- symbolic substitution
- identity sliding
- in-between state emphasis
- figure-pattern transitions

### Important note
This style becomes far more itself when generated as transitions between symbolic states rather
than isolated polished endpoints.

## Good generative priorities

If building a procedural system, it should prioritize:

1. symbol emergence
2. mutation continuity
3. tonal friction
4. contamination logic
5. anti-polish simplification
6. figure-pattern exchange
7. palette specificity
8. repetition with variation

## Recommended system strategy

### Strategy A — modular symbolic assembly
Build compositions from:
- a main glyph
- a contaminant glyph
- an optional cute motif cluster
- an optional abstract field
- a tone modifier

### Strategy B — mutation pass system
Start with a clean symbol or creature, then apply passes:
1. body glyph injection
2. Sweet Corruption
3. mock-esoteric framing
4. contour instability
5. hallucination-field integration
6. anti-polish pass

### Strategy C — emergence from abstraction
Begin with abstract field logic, then procedurally reveal:
- eyes
- mouths
- hands

- heads
- little creatures
- fake ritual symbols

This is especially good for shader-based or loop-based outputs.

## Recurrence and repetition

The style likes repeated motifs, but repetition should behave like:
- swarm logic
- symbolic chorus
- recursive implication
- dream repetition
- fake ritual insistence

Avoid repetition that feels:
- wallpaper-like
- lifeless
- purely decorative

## LSD-hallucination relation in systems terms

If the system needs a parameter for LSD-adjacent behavior, it should **not** mean:
- more rainbow

- more fractals
- more glowing nonsense

It should mean:
- faster symbolic reassignment
- increased figure-pattern instability
- more layered identity states
- more significance density
- greater chance of partial emergence
- higher metamorphic continuity

In system terms, “LSD logic” is about:
**semantic instability under strong visual pressure**

That is the useful translation.

## System failure warnings

### Weak system
- only palette changes
- only motif randomization
- no symbolic roles
- no motion logic
- no contamination

### Overbuilt system
- too many knobs
- too much polish bias
- too much realism
- too much symmetry
- too much coherent elegance

### Correct system
- simple motif library
- clear role logic
- mutation rules
- tone controls
- anti-drift gates
- emergence behaviors

## Final rule

Garbage Enlightenment should be generated like:
**a rude symbolic ecosystem where cute motifs, body parts, fake sacred signs, and abstract
hallucination fields keep infecting and becoming each other**
